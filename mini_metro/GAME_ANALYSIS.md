## 1. 핵심 게임 루프

```
[게임 시작]
    ↓
역이 랜덤 위치에 생성됨 (도형으로 구분)
    ↓
승객이 각 역에 스폰 — 목적지 도형 표시
    ↓
플레이어가 역들을 드래그로 노선 연결
    ↓
열차가 노선 따라 자동 왕복 운행
    ↓
열차가 해당 도형 역에 승객 하차
    ↓
주기마다 새 자원 지급 (노선/열차/객차)
    ↓
시간 경과 → 역 증가 → 승객 폭증
    ↓
역 하나라도 과부하 → 게임오버
    ↓
[최종 점수: 총 이송 승객 수]
```

### 핵심 긴장감 구조
- **공급 부족**: 자원(노선, 열차)은 한정, 역은 무한 증가
- **트레이드오프**: 새 노선 추가 vs 기존 노선에 열차 추가
- **실시간 재편성**: 언제든 노선을 지우고 다시 그을 수 있음

## 2. 엔티티 목록

| 엔티티 | 역할 | 플레이어 조작 여부 |
|--------|------|-----------------|
| `Station` | 승객이 기다리는 지점 | ❌ (자동 생성) |
| `Passenger` | 목적지로 이동하려는 개체 | ❌ (자동 행동) |
| `Line` | 역을 연결하는 노선 | ✅ (드래그로 편집) |

| `Carriage` | 열차에 추가하는 객차 | ✅ (자원으로 배치) |
| `Tunnel/Bridge` | 강/장애물 위를 잇는 연결 | ✅ (자원으로 배치) |

## 3. Station (역)

### 3.1 도형 종류
```
기본 도형 (초반부터 등장):
  ● 원 (Circle)
  ▲ 삼각형 (Triangle)
  ■ 사각형 (Square)

희귀 도형 (중반 이후 등장):
  ✦ 별 (Star)
  ✚ 십자 (Cross / Plus)
  다이아몬드, 오각형... (맵마다 다름)
```

### 3.2 역 생성 규칙
- **시작 시**: 원·삼각·사각 각 1개씩, 총 3개로 시작
- **생성 타이밍**: 일정 시간 간격으로 랜덤 위치에 추가
- **생성 간격**: 게임 진행할수록 점점 짧아짐 (난이도 증가)
- **위치 제약**: 강/물 위에는 생성 안 됨, 맵 경계 내부

### 3.3 역 과부하 시스템
```
정상 상태: 승객 수 < 임계값 (기본 6명)
경고 상태: 승객 수 ≥ 임계값 → 빨간 원 타이머 표시 시작
           타이머: 약 3~5분 (설정마다 다름)
게임오버: 타이머 만료
```

### 3.4 역 속성
```python
Station:
  - shape: ShapeType       # 역의 도형 타입
  - position: (x, y)       # 화면 좌표
  - passengers: List       # 대기 승객 목록
  - max_capacity: int      # 기본 6명 (업그레이드 가능)
  - overflow_timer: float  # 과부하 타이머 (None이면 정상)
  - connected_lines: List  # 연결된 노선 목록
```

## 4. Passenger (승객)

### 4.1 승객 목적지 규칙
- 자신이 **있는 역의 도형과 다른** 도형 역이 목적지
- 예: 원 역에 스폰된 승객 → 삼각 또는 사각 역으로 감
- 목적지 도형은 **아이콘으로 몸에 표시**됨
- 환승 가능: 목적지까지 여러 노선 갈아타도 됨

### 4.2 승객 승하차 로직
```
열차 도착 → 열차 목적지 도형 승객 우선 승차
           → 열차 용량 초과 시 나머지는 대기
열차 출발 → 이 역이 목적지인 승객 하차
           → 하차 후 더 이상 존재하지 않음 (카운트 +1)
```

### 4.3 환승 (Transfer) 로직
```
A 노선 열차 탑승 → B 역 도착 (환승 가능 역)
→ 목적지까지 B 노선이 더 유리하면 하차 후 대기
→ B 노선 열차 탑승 → 최종 목적지 하차
```
> ⚠️ 원작에서는 승객이 최단 경로를 자동으로 탐색함 (BFS/Dijkstra)

### 4.4 승객 속성
```python
Passenger:
  - destination_shape: ShapeType  # 가고 싶은 역의 도형
  - current_station: Station      # 현재 대기 중인 역
  - current_train: Train          # 탑승 중인 열차 (None이면 대기)
  - waiting_time: float           # 대기 시간 (통계용)
```

## 5. Line (노선)

### 5.1 노선 생성/편집
```
드래그 동작:
  - 역에서 드래그 시작 → 다른 역까지 드래그 → 놓으면 연결
  - 기존 노선 위에서 드래그 → 역 추가/경로 변경
  - 기존 노선 끝 구간을 반대 방향으로 드래그 → 구간 삭제
  - 노선 전체 삭제 버튼 (색상 패널에서)
```

### 5.2 노선 속성
```python
Line:
  - color: Color           # 노선 색상 (Red/Blue/Green/Yellow/Purple...)
  - stations: List[Station] # 연결된 역 순서 (순환 가능)
  - trains: List[Train]     # 이 노선에 배치된 열차들
  - is_loop: bool           # 순환 노선 여부
  - segments: List[Segment] # 역 간 선분 (터널 포함 여부)
```

### 5.3 노선 색상 (최대 노선 수)
| 색상 | 초기 제공 | 추가 획득 |
|------|----------|----------|
| 빨강 | ✅ 시작부터 | - |
| 파랑 | ✅ 시작부터 | - |
| 초록 | ✅ 시작부터 | - |
| 노랑 | ❌ | 자원으로 획득 |
| 보라 | ❌ | 자원으로 획득 |
| 주황 | ❌ | 자원으로 획득 |
> 최대 노선 수: **6~7개** (맵마다 다름)

### 5.4 강/장애물 처리
- 강을 건너는 구간은 **터널/다리** 자원 소모
- 자원 없으면 강을 건너는 노선 불가
- 강 위를 지나는 선분은 점선 또는 다른 시각 표현

## 6. Train (열차)

### 6.1 열차 이동 로직
```
노선의 첫 역 → 순서대로 다음 역 이동 → 마지막 역 도달
→ (순환 노선) 첫 역으로 돌아감
→ (왕복 노선) 반대 방향으로 되돌아감
```

### 6.2 역 정차 로직
```
역 도착 → [하차] 이 역이 목적지인 승객 내림
        → [승차] 열차 빈 공간만큼 대기 승객 태움
              → 태울 때 목적지 방향 고려 (단, 원작은 단순 선착순 + 공간 있으면 탑승)
        → 정차 시간: 승객 수에 비례 (약 0.5~1초)
        → 출발 → 다음 역으로
```

### 6.3 열차 속성
```python
Train:
  - line: Line              # 소속 노선
  - position: float         # 노선 상 위치 (0.0 ~ 1.0)
  - direction: int          # +1(정방향) or -1(역방향)
  - speed: float            # 이동 속도 (픽셀/초)
  - passengers: List        # 탑승 승객 목록
  - capacity: int           # 기본 6명 (객차 추가 시 +6)
  - carriages: int          # 추가 객차 수 (0~3)
  - state: str              # "moving" | "boarding" | "waiting"
  - dwell_timer: float      # 정차 타이머
```

### 6.4 열차 용량
| 상태 | 용량 |
|------|------|
| 기본 | 6명 |
| 객차 1개 추가 | +6 → 12명 |
| 객차 2개 추가 | +6 → 18명 |
| 객차 3개 추가 | +6 → 24명 |

## 7. 자원 시스템 (Resource)

### 7.1 자원 지급 타이밍
- **게임 내 1주일 주기**로 자원 지급 (약 실시간 3~5분)
- 현재 점수(이송 승객 수)에 따라 보너스 지급

### 7.2 자원 종류
| 자원 | 설명 | 전략적 가치 |
|------|------|------------|
| **열차 (Train)** | 노선에 열차 1대 추가 | 운행 빈도 증가 |
| **객차 (Carriage)** | 기존 열차에 객차 추가 | 단일 열차 용량 증가 |
| **노선 (Line)** | 새 노선 색상 1개 추가 | 네트워크 확장 |
| **터널 (Tunnel)** | 강을 건너는 연결 1개 | 맵 장애물 극복 |

### 7.3 자원 전략
```
객차 > 열차 > 노선 > 터널 (일반적 우선순위)
- 객차: 이미 바쁜 노선의 처리량을 즉시 늘림
- 열차: 긴 노선의 배차 간격 줄임
- 노선: 완전히 새로운 루트 개척
- 터널: 맵에 강이 많을 때 필수
```

## 8. 게임오버 조건

```
단 하나의 역이라도:
  passengers.count >= max_capacity (6명)
  AND overflow_timer 만료 (약 3~5분 방치)

→ 즉시 게임오버
→ 최종 점수 (이송 승객 수) 표시
→ 리플레이 또는 메인 메뉴
```

## 9. 점수 시스템

```python
score += 1  # 승객이 목적지 역에 하차할 때마다

# 게임 종료 시:
final_score = total_passengers_delivered
high_score = max(high_score, final_score)
```

## 10. 맵 & 환경

### 10.1 원작 맵 목록 (일부)
| 맵 | 특징 |
|----|------|
| 런던 | 기본 튜토리얼 맵, 강(템즈) 있음 |
| 파리 | 환형 구조 |
| 뉴욕 | 격자형 |
| 도쿄 | 복잡한 노선, 빠른 승객 증가 |
| 오클랜드 | 섬 지형, 터널 필수 |

### 10.2 맵 구성 요소
```
배경: 단색 (흰/검정/회색) + 강/물 구역
강: 노선 연결 시 터널 자원 소모
경계: 맵 밖으로 역 생성 안 됨
```

### 10.3 클론 구현 시 맵
- **Phase 1**: 배경 단색, 강 없음 (가장 단순)
- **Phase 2**: 강 추가 (수평/수직 강)
- **Phase 3**: 실제 도시 윤곽 기반 맵


### 11.2 UI 요소 상세
| 요소 | 위치 | 기능 |
|------|------|------|
| 점수 | 상단 중앙 | 이송 승객 수 실시간 표시 |
| 일시정지 버튼 | 상단 좌 | 정지 (이때 노선 편집 가능) |
| 속도 버튼 | 상단 우 | 1x / 2x 배속 전환 |
| 노선 패널 | 하단 | 각 노선 선택/삭제, 활성 노선 표시 |
| 자원 창고 | 하단 | 보유 열차/객차/터널 수량 표시 |
| 주간 보상 팝업 | 전면 | 주기마다 자원 선택 UI |
| 게임오버 화면 | 전면 | 최종 점수, 리플레이 버튼 |

### 11.3 노선 편집 인터랙션 (중요!)
```
[드래그로 노선 그리기]
  Mouse Down on Station → drag → Mouse Up on Station
  = 두 역 사이에 노선 선분 추가

[노선에 역 추가]
  기존 노선의 끝에서 드래그 → 새 역까지

[구간 삭제]
  노선 끝에서 안쪽으로 드래그 → 마지막 역 제거

[전체 노선 삭제]
  하단 패널의 해당 노선 색상 길게 누르기
```

## 13. 구현 난이도 분류

### 🟢 쉬움 (Phase 1에서 구현)
- 역 생성 & 도형 렌더링
- 노선 드래그로 연결 (직선 선분)
- 열차 기본 이동 (선분 따라 왕복)
- 기본 승객 스폰 & 하차
- 점수 카운터
- 게임오버 판정

### 🟡 중간 (Phase 2에서 구현)
- 승객 최단경로 탐색 (BFS)
- 열차 정차 & 승하차 애니메이션
- 객차 추가 시스템
- 주간 자원 지급 팝업
- 노선 편집 (중간에 역 삽입/삭제)
- 과부하 타이머 UI

### 🔴 어려움 (Phase 3에서 구현)
- 강/장애물 + 터널 시스템
- 환승 최적 경로 (멀티홉 BFS)
- 노선 루프(순환) 처리
- 실제 도시 맵 윤곽
- 모바일 터치 최적화
- 사운드 시스템

## 요약 — 최소 구현 범위 (MVP)

```
엔티티:
  ✅ Station (역): 도형 3종, 생성, 과부하 판정
  ✅ Passenger (승객): 스폰, 목적지 도형, 하차
  ✅ Line (노선): 드래그 연결, 최대 3개
  ✅ Train (열차): 노선 왕복, 승하차

게임 흐름:
```
