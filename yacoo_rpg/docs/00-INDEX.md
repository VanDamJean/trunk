# Yacoo RPG — 작업 핸드오프 문서 (INDEX)

> 이 문서 묶음은 **외부 AI(GPT / GLM 등)에게 작업을 맡기기 위한 핸드오프 명세**입니다.
> 작업을 받는 AI는 이 `docs/` 폴더의 문서만 읽고도 맥락 없이 작업을 시작할 수 있어야 합니다.

## 한 줄 요약

기본 야추(Yahtzee) 주사위 룰을 베이스로, **하비(Habby)의 "딸깍 다이스"** 스타일 모바일 RPG를 만든다.
React + Vite + TypeScript 웹 프로토타입(`yacoo_rpg/`)이 이미 약 70% 골격을 갖춘 상태이며, 이 문서들은 거기서부터 "딸깍 다이스화" 하기 위한 작업 명세다.

## 핵심 방향 (의사결정 완료)

이 세 가지는 이미 결정되었다. 임의로 바꾸지 말 것.

1. **전투 모델 = 수동 턴제.** (※ 초기엔 하이브리드로 정했으나 번복됨.) 시간 기반 오토배틀을 폐기하고, 매 턴 플레이어가 주사위를 굴려 족보로 공격하는 순수 턴제로 간다. 평타는 자동이 아니라 매 턴 선택지로만 남긴다. **최종 사양은 `06-COMBAT-MANUAL-TURN-SPEC.md`.**
2. **성장 구조 = 런 + 메타 둘 다.** 한 판(런) 안에서는 로그라이크(챕터/노드 진행, 특성·보물 수집, 사망 시 런 종료), 런 밖에서는 영구 메타 성장(코인, 장비 강화, 해금).
3. **타깃 = 추후 안드로이드 포팅.** 지금은 웹에서 기획·검증하되, 나중에 Kotlin/Compose 포팅을 염두에 두고 설계한다(워크스페이스의 다른 `*-android` 프로젝트들과 동일 패턴).

## 작업을 받는 AI에게 주는 지침

- **코드 작성 전 반드시 `05-CODING-GUIDELINES.md`를 먼저 읽을 것.** 특히 "`game/` 폴더 순수성 유지"와 "RNG 주입" 규칙은 안드로이드 포팅의 전제 조건이다.
- **마일스톤 순서대로(M1 → M5) 진행할 것.** 각 마일스톤은 독립적으로 빌드·테스트가 통과해야 한다.
- **한 마일스톤이 끝날 때마다** `npm run build`, `npm test -- --run`, `npm run qa` 세 개를 모두 통과시킬 것(자세한 명령은 `README.ko.md` 참고).
- **기존 테스트를 깨뜨리지 말 것.** 동작을 바꿔야 한다면 테스트도 같이 갱신하고, 갱신 이유를 커밋 메시지/PR 설명에 남길 것.
- **외부 게임 자산(딸깍 다이스 등)의 이미지/사운드/코드를 복사하지 말 것.** 메커니즘만 참고한다. 그림은 프로젝트 자체 CSS/SVG 컴포넌트로 만든다(`src/components/ui.tsx`, `src/components/GameArt.tsx`).
- **UI 문구는 현재 영어로 통일되어 있다.** 새로 추가하는 UI도 일관성을 위해 영어로 두되, 필요 시 별도 작업으로 한글화한다(코드 식별자는 항상 영어).
