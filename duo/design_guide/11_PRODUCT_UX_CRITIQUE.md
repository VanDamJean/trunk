## 2. UX 문제점

### 문제 A: 첫 사용 온보딩 없음

**현상**: 앱 첫 진입 시 바로 홈 화면. 앱이 뭔지, 어떻게 쓰는지 설명 없음.
**영향**: 이탈률 높음. "학습 시작" 버튼을 누르면 뭐가 나오는지 예측 불가.
**권장**: 3장짜리 온보딩 카드 (앱 소개 → 목표 설정 → 언어 선택). P1.

### 문제 B: 레슨 중 이탈 시 학습 손실 불안

**현상**: ✕ 클릭 → `confirm()` 대화상자 "진행 상황은 저장됩니다" 표시.
**문제**: 브라우저 네이티브 confirm은 앱 느낌을 깨뜨림. 저장 범위도 불명확 (어디까지 저장되는지).
**권장**: 커스텀 바텀 시트 확인 다이얼로그 + "저장된 문제까지 기록됩니다" 명시. P2.

### 문제 C: 빈칸 채우기에서 힌트가 너무 직관적

**현상**: `힌트: e_______l` — 첫글자+끝글자+글자수가 전부 노출.
**문제**: 힌트가 사실상 답을 알려줌. 학습 효과 감소.
**권장**: 1차 시도에는 힌트 숨기고, 틀리면 힌트 표시하는 2단계 구조. P2 (quizEngine 수정 필요).

### 문제 D: 리그 봇이 명확히 봇임

**현상**: 리더보드 이름이 이모지+한국어 닉네임. LP가 매주 같은 패턴.
**문제**: 사용자가 봇임을 알아채면 경쟁 동기 소실.
**권장**: 이름 풀 다양화, LP 분포를 정규분포 기반 랜덤으로. P2 (league.js 수정 필요).

### 문제 E: 광고 보상이 가짜

**현상**: "광고 보고 +30 LP 받기" 버튼이 있지만 실제 광고 SDK 없음. 클릭하면 바로 LP 지급.
**문제**: 앱 스토어에 올리기 전에 해결 필요. 현 단계에서는 OK.
**권장**: 실제 광고 연동 전까지 "보너스 LP 받기"로 표현 변경 고려. P2.

### 문제 F: 학습 진행 상황이 홈에만 있음

**현상**: 오늘의 학습 진행률이 홈 화면에만 표시. 레슨 중에는 전체 중 몇 번째인지만 보임.
**문제**: 레슨 완료 후 "오늘 목표 달성까지 얼마나 남았는지" 바로 안 보임.
**권장**: 레슨 완료 화면에 "오늘 목표 7/15 (8개 더!)" 같은 맥락 추가. P1.

### 문제 G: 단어장이 수동 탐색용이고 학습과 연결 안 됨

**현상**: 단어장에서 개별 단어를 보고 발음을 들을 수 있지만, "이 단어를 복습하기" 같은 액션이 없음.
**문제**: 단어장이 참조용에 그침.
**권장**: 단어 상세 모달에 "이 단어 복습하기" 버튼 추가 → 해당 단어만 포함된 미니 세션 시작. P2 (scheduler 수정 필요).

### 문제 H: 레이팅 버튼의 의미가 불명확

**현상**: "모름 / 어려움 / 알겠음 / 쉬움" + interval 표시.
**문제**: 사용자가 4개 중 뭘 눌러야 할지 고민. "알겠음"과 "쉬움"의 차이가 직관적이지 않음.
**권장**: FSRS의 4-rating을 유지하되, 설명 텍스트를 좀 더 구체적으로 ("전혀 기억 안 남" / "겨우 생각남" / "기억남" / "너무 쉬움"). P1.

## 3. 화면별 UX 개선 포인트

### Home

| 현재 | 개선 |
|---|---|
| 인사말이 정적("오늘도 단어를 배워볼까요?") | 스트릭 기반 동적 문구 ("3일 연속! 오늘도 이어가요") |
| Quick Action 2개만 (복습/단어장) | "오늘의 틀린 단어" 같은 맥락형 퀵 액션 추가 |
| 언어 셀렉트가 우상단에 작게 | 학습 언어가 앱의 핵심 맥락인데 시각적 비중 낮음 |

### Lesson

| 현재 | 개선 |
|---|---|
| 프로그레스 바 + "3/8" 텍스트 | 충분함. 변경 불필요 |
| 콤보 피드백이 텍스트로만 ("🔥 3연속 정답!") | 콤보 카운터 UI (숫자가 커지는 시각 효과) |
| 매칭 퀴즈가 항상 4쌍 | 난이도 조절 (레벨에 따라 3~6쌍) — quizEngine 수정 필요 |

### Wordbook

| 현재 | 개선 |
|---|---|
| 숙달도 뱃지가 텍스트(새/학습/복습/숙달) | 색상 코딩은 좋으나 뱃지 안의 글자가 너무 작음 (0.7rem) |
| 카테고리 필터만 | 숙달도 필터 추가 ("아직 모르는 단어만 보기") |

### League

| 현재 | 개선 |
|---|---|
| 리그 이름이 영어 ("Bronze League") | 앱 UI가 한국어인데 리그만 영어. "브론즈 리그"로 통일 |
| 주간 결과 카드가 한 줄 | 승급 시 축하 이모션, 강등 시 격려 메시지 |

## 4. Figma 리디자인 시 반영 우선순위

| 우선순위 | 항목 | 화면 | 비고 |
|---|---|---|---|
| 1 | 레이팅 버튼 라벨 개선 | Lesson, Review | 텍스트만 변경. 디자인에서 확정 후 코드 적용 |
| 2 | 홈 인사말 동적화 설계 | Home | 스트릭/완료 상태에 따른 문구 variants |
| 3 | 레슨 완료에 오늘 목표 맥락 추가 | Lesson Complete | 프레임에 "7/15 달성" 영역 설계 |
| 4 | 리그 이름 한국어화 | League | "Bronze → 브론즈" 매핑 테이블 확정 |
| 5 | 단어장 숙달 필터 | Wordbook | 필터 칩에 "미학습" 등 추가 |
| 6 | 온보딩 3장 | 새 화면 | 완전 새 디자인 필요 |
| 7 | 콤보 카운터 UI | Lesson | 기존 텍스트 → 시각 요소 |

## 5. 건드리면 안 되는 UX

| 항목 | 이유 |
|---|---|
| FSRS 4-rating 체계 | 알고리즘 핵심. 3단계나 5단계로 바꾸면 스케줄링 깨짐 |
| 하루 15문제 기본값 | 행동과학적으로 검증된 양. 사용자가 변경 가능하게는 OK |
| 오프라인 우선 | 서버 의존으로 바꾸면 핵심 가치 훼손 |
| XP + LP 분리 | XP는 개인 성장, LP는 경쟁. 합치면 인센티브 구조 무너짐 |
| 퀴즈 타입 자동 결정 | 사용자가 퀴즈 타입을 고르는 건 학습 효과를 떨어뜨림 |
