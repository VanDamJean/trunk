# Gemsin 디자인 고정 스펙

## 1) 작업 범위

- 구현 대상 아님: 앱 코드/웹 코드 작성 금지
- 현재 문서는 **디자인 요소 정리 및 보존 규칙**만 다룬다
- 목표: 후속 앱 개발 시 Figma 원본과 동일한 UI를 유지하기 위한 기준 문서

## 2) 디자인 불변 규칙 (반드시 준수)

- 버튼 크기, 버튼 모서리, 버튼 색상 변경 금지
- 텍스트 크기, 두께, 줄간격, 정렬 변경 금지
- 카드 폭/높이, 여백, 패딩, 간격 변경 금지
- 아이콘 크기/위치/색상 변경 금지
- 하단 탭바 높이/아이콘 위치/활성 색상 변경 금지
- 상태바, 상단 헤더, CTA 위치 변경 금지
- 원본 이미지/일러스트/픽셀아트 변경 금지
- 화면 기본 비율은 `375x812`를 기준으로 유지

## 3) 확인된 화면 성격 (샘플 검증)

- `1:46608`(Welcome 1로 확인)
  - 레트로 픽셀아트 + 다크 카드 + 라임 컬러 CTA
  - 메시지: 게임 추천/진행 저장/성장 추적
- `1:46307`(홈 대시보드로 확인)
  - 상단 잔액/프로필, 검색바, 게임 카드 그리드, 하단 탭바

## 4) 전체 화면 인덱스

- 전체 `375x812` 프레임(clip/그룹 포함): `frame_index_375x812.csv` (164개)
- 실제 화면 후보(`name=Group`): `screen_frames.csv` (40개)
- 화면 구현 시 `screen_frames.csv`의 `id`를 우선 기준으로 사용

## 5) 현재 수집 제한 사항

- Figma MCP Starter 호출 제한에 도달하여 추가 자동 캡처는 중단됨
- 따라서 이 문서는 구조/프레임 중심으로 먼저 고정함
- 이후 호출 제한 해제 시, 각 화면의 정확한 컬러 토큰/폰트/반경/스트로크 값을 추가 보강 필요

## 6) 후속 앱 개발 전 체크리스트

- Figma Inspect에서 각 주요 컴포넌트의 실제 수치 잠금
  - 색상(HEX/RGBA)
  - 폰트(패밀리/크기/굵기/행간)
  - 반경/보더/그림자
  - 오토레이아웃 간격/패딩
- `screen_frames.csv` 기준으로 화면 순서와 네비게이션 흐름 확정
- 공통 컴포넌트 정의
  - 버튼(Primary/Secondary/On-dark)
  - 카드(게임 카드/통계 카드/알림 카드)
  - 입력(검색 필드)
  - 탭바(기본/활성 상태)
- 구현 단계에서도 "디자인 불변 규칙"을 린트 규칙처럼 적용
