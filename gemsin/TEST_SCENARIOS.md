# Gemsin Mock Test Scenarios

## 목적

- 백엔드 없이 프론트 목업의 주요 클릭 플로우를 빠르게 검증한다.

## 시나리오 A: 온보딩 진입/스킵

1. 초기 화면이 `onboarding-1`인지 확인
2. `❯` 버튼으로 `onboarding-2`, `onboarding-3` 순서 이동 확인
3. 각 화면의 `Skip` 클릭 시 `home-screen` 진입 확인

## 시나리오 B: 홈 게임 카드와 상세 모달

1. 홈에서 게임 카드 아무거나 클릭
2. 상세 모달에 게임명/설명/모드/난이도 표시 확인
3. 모달의 `Start Game` 클릭 시 mock 알림 확인
4. `Close` 또는 바깥 영역 클릭 시 모달 닫힘 확인

## 시나리오 C: 리더보드 탐색

1. `Leaderboard` 탭 이동
2. 검색창에 `you` 입력 시 해당 row만 노출되는지 확인
3. `Sort: High` 버튼 클릭 시 `Sort: Low`로 토글 및 순서 변경 확인
4. `Weekly` 버튼 클릭 시 `Monthly` 토글 확인

## 시나리오 D: 티켓/크레딧 플로우

1. `Credit & Ticket` 탭 이동
2. `Claim Daily +5` 클릭 후 크레딧 증가 확인
3. Pack 버튼들 클릭 후 크레딧이 각각 +50/+250/+700 증가하는지 확인

## 시나리오 E: 친구/히스토리 필터

1. `Teman` 탭에서 검색어 입력 시 친구 필터링 확인
2. `Follow` 버튼 토글 시 `Following` 상태 변경 및 알림 확인
3. `History` 탭에서 `Today`/`Yesterday` 필터 클릭 시 리스트 필터링 확인

## 체크 포인트

- 탭 이동 중 화면 깨짐/오버플로우 없음
- 버튼 중복 클릭 시 JS 에러 없음
- mock 상태 변경(크레딧/팔로우/필터)이 즉시 반영됨
