# Arrow Puzzle - 현재 구현 상태 및 향후 개발 레포트

## 1. 완료된 작업 내용 (Completed Features)

현재 구현은 이미지 기반 마스크 생성, 랜덤 경로 기반 퍼즐 생성, Undo, JSON Export를 포함한 브라우저용 프로토타입입니다. 복잡한 이미지나 밀집된 마스크에서는 생성 실패 또는 `1cell` 폴백이 발생할 수 있습니다.

### ✅ 1) 이미지 업로드 기반 자동 맵 생성 (Image to Puzzle)
- **로직 구현:** 하단 `Image Upload` 버튼을 통해 브라우저에서 이미지 파일을 업로드할 수 있습니다.
- **이미지 분석 알고리즘:** 가상 캔버스를 통해 투명 이미지는 alpha 값을 기준으로, 불투명 이미지는 밝기 기반 Otsu 임계값으로 0과 1의 타겟 마스크를 추출합니다.
- **오토 리사이징:** 업로드된 이미지의 비율을 유지하면서 현재 `Grid Max` 범위 안에서 마스크 크기를 계산합니다.
- **제약:** 모든 이미지에서 의도한 실루엣이나 고품질 squiggly 퍼즐을 보장하지는 않습니다. 생성 실패 시 기본 하트 마스크 또는 `1cell` 방식으로 폴백할 수 있습니다.

### ✅ 2) 다이내믹 그리드 스케일링 (Dynamic Scaling)
- **그리드 자동 조절:** `Grid Max` 슬라이더(8~40)를 통해 그리드의 촘촘함을 결정할 수 있습니다. 이미지를 올리면 캔버스의 격자(CELL_SIZE)와 화살표의 두께(LINE_WIDTH)가 자동으로 스케일링되어 계산됩니다.
- **화살촉 비율 수정:** 선이 얇아져도 화살촉(`Cutout` 효과 적용)의 크기가 비율에 맞게 정확히 파이도록 수학 공식을 수정했습니다.

### ✅ 3) JSON 레벨 데이터 추출 (Export System)
- **저장 기능:** `Export JSON` 버튼을 누르면, 현재 맵의 화살표 좌표(`pts`)와 색상(`color`) 데이터를 `arrow_level.json` 파일로 다운로드합니다.
- 이를 통해 생성기가 만들어낸 우연한 예술작품(맵)을 영구 보관할 수 있습니다.

## 2. 향후 과제 및 남은 스텝 (Remaining Roadmap)

현재 코어 프로토타입은 동작하지만, 상용 퀄리티의 퍼즐 게임앱이 되기 위해서는 아래 작업이 남아 있습니다.

1. **아이템 시스템 및 경제 구조**
   - 힌트(Hint), 셔플(Shuffle) 기능 구현.
   - 우측 하단에 아이템 인벤토리 UI를 만들고, 클리어 시 코인을 지급하여 상점에서 아이템을 사게 만드는 루프 구현.
2. **스테이지 프로그레션 (로비 및 저장 기능)**
   - Export로 모아둔 `arrow_level.json` 맵 50개를 묶어 Level 1 ~ 50까지 구성.
   - 홈 화면(Lobby)을 만들고, `localStorage`를 통해 유저가 몇 단계까지 클리어했는지 진척도(Save) 구현.
3. **비주얼 폴리싱 강화**
   - 현재 단색 화살표에 그라데이션 질감 추가.
   - 배경 타일 격자에 부드러운 네온 네트(Neon Net) 효과나 어두운 테마 스킨(Skin) 상점 추가.
