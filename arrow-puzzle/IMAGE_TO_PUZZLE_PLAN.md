# 이미지 기반 퍼즐 자동 생성기 (Image-to-Puzzle Generator) 기획서

## 1. 개요 (Overview)
외부 이미지(PNG, SVG 등)를 불러와서, 해당 이미지의 실루엣 형태를 띄는 'Arrow Puzzle' 레벨 데이터를 자동으로 생성하는 알고리즘.
수동으로 맵을 디자인하는 한계를 벗어나 무한한 스테이지 생성을 가능하게 함.

## 2. 현재 구현된 변환 알고리즘
현재 `main.js`는 이미지를 작은 그리드 마스크로 변환한 뒤, 마스크 내부의 빈 칸을 무작위 경로로 채우고 각 경로의 머리 방향에서 탈출 가능한지 검사하는 방식으로 레벨을 생성한다. `mixed`/`squiggly` 생성이 밀집된 마스크에서 실패하면 `1cell` 생성으로 폴백한다.

초기 기획의 **역순(Reverse) 조립 알고리즘**은 아직 별도 구현되어 있지 않다. 따라서 현재 구현은 생성 단계에서 탈출 방향을 검사하고 실패 시 폴백하는 방식이며, 모든 이미지/모든 형태에 대해 수학적으로 100% 클리어 가능성을 보장한다고 표현하지 않는다.

### Phase 1: 이미지 분석 (Grid Mapping) - 구현됨
1. 보이지 않는 가상 Canvas(Offscreen Canvas)에 이미지를 렌더링.
2. 게임의 그리드 사이즈(예: 20x20)에 맞게 픽셀 데이터를 샘플링(`getImageData`).
3. 색상이 있는 부분은 `1`(블록 위치), 투명한 부분은 `0`(빈 공간)으로 이루어진 2D 배열 형태의 **목표 실루엣(Target Mask)** 생성.

### Phase 2: 현재 레벨 생성 방식 - 구현됨
목표 실루엣 내부의 빈 칸을 채울 때마다 랜덤 시작점과 랜덤 길이의 경로를 만들고, 경로의 마지막 지점에서 화면 밖으로 뻗는 방향 중 기존 블록과 충돌하지 않는 방향을 찾는다.
1. 목표 실루엣 내부의 모든 빈 칸이 채워질 때까지 반복 루프 시작.
2. 비어 있는 칸 하나를 시작점으로 선택하고 `1cell`, `squiggly`, `mixed` 타입에 따라 경로 길이를 정한다.
3. 이미 채워진 칸을 피하면서 상하좌우 인접 칸으로 경로를 확장한다.
4. 경로의 머리 지점에서 네 방향 중 기존 배치와 충돌하지 않고 밖으로 나갈 수 있는 방향을 찾는다.
5. 배치가 막히면 재시도하고, 반복 실패 시 `1cell` 생성으로 폴백한다.

### Phase 3: 향후 옵션 - Reverse Injection
퍼즐을 푸는 행동(블록이 밖으로 나감)을 역재생하여, 블록이 밖에서 안으로 밀고 들어오는 방식으로 퍼즐을 조립하는 아이디어다. 이 방식은 이론적으로 더 강한 클리어 가능성 보장을 줄 수 있지만, 현재 코드에는 구현되어 있지 않다.

## 3. 구현된 생성 타입
* **1-Cell Blocks**
  - 꼬리 없이 머리(화살표)로만 이루어진 딱 1칸짜리 블록들로 이미지를 채우는 방식. 
  - 알고리즘 난이도가 낮고 연산이 빠름. 틱톡 유행 스타일.
* **Multi-Cell Squiggly Paths**
  - 3~10칸의 길이를 가진 꺾이는 선(Path)을 무작위로 욱여넣는 방식. 
  - 더 복잡한 실루엣을 만들 수 있지만, 현재 구현에서는 실패 시 1-Cell 방식으로 폴백할 수 있다.
* **Mixed**
  - 1-Cell과 Multi-Cell 경로를 섞어 생성한다.
