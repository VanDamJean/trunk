# 말미잘 방치형: 뿅 회사 고도화 설계도

## 0. 한줄 판정

야 시시포스, 껍데기랑 컨셉은 이미 약 빨고 잘 뽑혔다. 지금 필요한 건 더 예쁜 버튼이 아니라, 유저가 3분 동안 `뿅! 내가 천재 CEO다` 하다가 5분 차에 `어? 왜 회사가 해조류처럼 안 움직이지?` 하고 광고나 연구를 누르게 만드는 숫자 장벽이다.

현재 게임 기준점:

- 수동 클릭: `사장 말미잘 찰싹!` 1회당 최소 꿈틀밥 `+3`, 이후 현재 꿈틀밥 생산량 약 10초치만큼 함께 증가
- 현재 생산자: `인턴 촉수`, `새우 중간관리자`, `조개 인플루언서 농장`, `수상한 달빛 호스`
- 현재 성장식: `Cost = Base x Growth^Owned`, 생산자별 성장률 `1.14~1.20`
- 목표 성장식: 핵심 생산 라인은 `Cost = Base x 1.15^Level`로 통일
- 현재 UI 뼈대: 상단 3자원 헤더, 회사식 무리수 미션 가로 카드, 하단 4탭(`노동 산호초`, `전리품 조개`, `허세 진주`, `언어 도깨비`)
- 현재 시간 장치: 전리품 조개 `5분`, 오프라인 보상 최대 `4시간`

## 1. 생산 라인 4티어 스케일업

### 1.1 계층 구조

| 티어 | 이름 | 회사 계급 | 생산 대상 | 잠금 의도 | 한줄 개그 |
|---:|---|---|---|---|---|
| 1 | 인턴 촉수 | 무급 실무자 | 꿈틀밥 직접 생산 | 시작 즉시 | 작은 팔들이 스프레드시트를 흔들다 간식이 떨어진다. |
| 2 | 새우 중간관리자 | 회의 소환사 | 꿈틀밥 직접 생산 | 1분 내 첫 목표 | 거품에 빨간 줄 치고 꿈틀밥이 늘었다고 보고한다. |
| 3 | 대게 지사장 | 지역 본부장 | 새우 자동 생산 | 4~6분 벽 돌파 장치 | 집게로 조직도를 찢더니 새우를 복사한다. |
| 4 | 고래 대주주 | 바다 재벌 | 대게 자동 생산 | 12분 이후 장기 목표 | 숨 한 번 쉬면 이사회가 물보라에 승인된다. |

핵심은 3티어부터 `자원`이 아니라 `하위 생산자 수량`을 만든다는 점이다. 유저가 직접 꿈틀밥만 보던 단계에서 조직도가 살아 움직이는 단계로 넘어간다.

### 1.3 체인 환산 규칙

| 항목 | 설계값 | 의도 |
|---|---:|---|
| 대게 1마리 생산량 | 새우 `0.015/s` | 약 66.7초마다 새우 1마리. 공짜처럼 보이지만 답답하게 달다. |
| 대게 10마리 생산량 | 새우 `0.15/s` | 약 6.7초마다 새우 1마리. 광고 버프 체감이 보이기 시작한다. |
| 고래 1마리 생산량 | 대게 `0.002/s` | 약 8.3분마다 대게 1마리. 장기 방치용 재벌 냄새. |
| 고래 10마리 생산량 | 대게 `0.02/s` | 약 50초마다 대게 1마리. 다음날 복귀 보상의 명분. |

주의: 하위 생산자 자동 생성은 정수로 끊지 말고 내부적으로 `fractionalProducerProgress`를 둔다. 겉으로는 `새우 +1`만 보여줘도, 내부 소수점은 계속 굴러야 유저가 손해 봤다는 느낌이 없다.

### 2.1 목표 감정 곡선

### 2.4 벽을 불쾌하지만 접지 않게 만드는 장치

| 장치 | 효과 | 노출 문구 | 제한 |
|---|---|---|---|
| 임원용 국수 팔 | 모든 생산량 `x1.25` | `회의실에서 팔이 25% 더 길어졌습니다.` | 허세 진주로 구매, 첫 5분 벽 추천 |
| 보상형 광고: 야근 국물 | 2분간 꿈틀밥 생산 `x2` | `광고 보고 합법 야근 국물 투입` | 10분당 1회, 중첩 불가 |
| 보상형 광고: 대게 결재도장 | 90초간 대게의 새우 생산 `x3` | `지사장이 도장을 난사합니다.` | 대게 1마리 이상 보유 시 노출 |
| 보상형 광고: 조개 퀵배송 | 진행 중 전리품 조개 시간 `-60초` | `조개가 택배기사에게 쫓깁니다.` | 조개 진행 중일 때만 |
| 미션 보상 | 벽 직전 소량 보정 | `회사식 무리수 완료` | 반복 파밍 불가 |

광고는 `안 보면 게임 못 함`이 아니라 `안 보면 2분 멍 때림` 정도가 맞다. 억지 과금 냄새보다 `이 미친 회사에 국물 한 번 붓자` 느낌이 더 오래 간다.

## 3. 상업용 기능 고도화 로드맵

### 3.1 웹에서 모바일 네이티브 앱 전환 패키징

현재 프로젝트는 Vite 웹앱이다. 네이티브 앱으로 가장 덜 아픈 길은 `Capacitor` 포장이다. React Native로 갈아타면 앱처럼 보이긴 하는데, 지금 잘 만든 병맛 웹 UI를 한 번 더 갈아엎는 대형 삽질이 된다.

| 단계 | 작업 | 산출물 | 체크포인트 |
|---:|---|---|---|
| 1 | Capacitor 도입 | `android/`, `ios/` 네이티브 프로젝트 | 웹 빌드 산출물 `dist`를 앱에 싣는 구조 확정 |
| 2 | 앱 ID/이름 확정 | 예: `com.bloopoffice.anemoneidle` | 스토어 출시 전 바꾸기 귀찮으니 여기서 못 박기 |
| 3 | 빌드 루프 정리 | `npm run build` -> `npx cap sync` -> Android Studio/Xcode 빌드 | 웹 수정과 네이티브 싱크 순서 문서화 |
| 4 | 저장소 마이그레이션 검토 | localStorage 유지 또는 네이티브 Preferences 이전 | 기존 웹 세이브를 앱에서 어떻게 볼지 결정 |
| 5 | Safe Area/폰트/진동 | 모바일 노치, 하단 내비, 햅틱 | 현재 430px 모바일 쉘과 하단 탭 충돌 확인 |
| 6 | 스토어 준비 | 아이콘, 스플래시, 개인정보 처리방침 | 광고 SDK 쓰는 순간 개인정보 문서 필수 |
| 7 | 릴리즈 빌드 | AAB/APK, iOS archive | 테스트 광고 ID와 실광고 ID 분리 |

공식 흐름 기준으로 Capacitor는 웹 빌드 후 `npx cap sync`로 `dist`를 네이티브 프로젝트에 복사하고 플러그인 의존성을 갱신한다. 즉, 이 게임의 첫 상업 버전은 `웹앱을 앱 껍데기에 봉인한 뒤 광고 SDK만 네이티브로 붙이는 방식`이 가장 현실적이다.

### 3.2 Rewarded Ad 버프 시스템 백단 설계

보상형 광고는 클라이언트에서 `광고 봤음`이라고 우기는 순간 바로 털린다. 출시형 구조는 Google AdMob 서버 측 검증(SSV)을 기준으로 잡는다.

```json
      "nameKo": "야근 국물",
      "rewardType": "productionMultiplier",
      "target": "plankton",
      "multiplier": 2,
      "durationSeconds": 120,
      "cooldownSeconds": 600,
      "unlock": "firstWallReached"
    },
    {
      "id": "crabStampedeStamp",
      "nameKo": "대게 결재도장",
      "rewardType": "producerSpawnMultiplier",
      "target": "shrimpManager",

```

### 3.3 Rewarded Ad 처리 플로우

| 순서 | 주체 | 처리 | 실패 시 |
|---:|---|---|---|
| 1 | 클라이언트 | `buffId`, `userId`, `nonce` 생성 후 광고 준비 | 광고 버튼 비활성, `조개가 아직 광고주랑 싸우는 중` 표시 |
| 2 | AdMob SDK | Rewarded Ad 표시 | 닫으면 보상 없음, 쿨다운 없음 |
| 3 | Google | SSV 콜백으로 백엔드 호출 | 콜백 지연 가능하므로 클라 즉시 보상 금지 |
| 4 | 백엔드 | Google 서명 검증, `transactionId/nonce` 중복 확인 | 검증 실패 로그, 보상 지급 안 함 |
| 5 | 백엔드 | 보상 원장 `ad_reward_ledger`에 기록 | 중복이면 성공 응답만 주고 재지급 없음 |
| 6 | 클라이언트 | 보상 상태 조회 또는 푸시성 갱신 | `광고 영수증 확인 중` 상태 유지 |
| 7 | 게임 | 버프 시작/시간 차감 | 앱 재시작 후에도 만료 시각 기준으로 복원 |

### 3.4 보상 원장 데이터 모델

```json
{
  "adRewardLedger": {
    "id": "uuid",
    "userId": "stable-player-id",
    "adNetwork": "admob",
    "adUnitId": "rewarded-production-boost",
    "buffId": "overtimeSoup",
    "nonce": "client-generated-single-use-token",
    "transactionId": "google-callback-transaction-id-or-derived-hash",
    "ssvSignatureVerified": true,
    "rewardGranted": true,
    "grantedAt": "server-time",
    "expiresAt": "server-time-plus-duration",
    "rawCallbackHash": "sha256-for-audit"
  }
}
```

반드시 지켜야 할 규칙:

- `nonce`는 1회용이다. 같은 `nonce`가 다시 오면 조개껍데기로 싸대기만 치고 보상은 없다.
- 보상은 서버 시간이 기준이다. 클라이언트 시간이 미래로 가도 고래 대주주가 타임머신을 타면 안 된다.
- 같은 광고 콜백은 여러 번 올 수 있다. 원장은 멱등 처리한다.
- 테스트 광고 ID와 실광고 ID를 환경별로 분리한다.
- `userId`가 아직 계정 기반이 아니면 임시 `installId`를 만들되, 나중에 계정 연동 시 병합 정책을 따로 둔다.

### 3.5 UI 배치 메모

현재 UI는 모바일 쉘에 상단 자원 헤더와 하단 탭이 고정되어 있다. 광고 버튼은 새 탭을 만들지 말고 기존 흐름에 기생시키는 편이 낫다.

| 위치 | 버튼 | 노출 조건 | 이유 |
|---|---|---|---|
| 노동 산호초 생산자 리스트 상단 | `야근 국물 광고 보기` | 5분 벽 감지 또는 대게 가격의 60% 미만 보유 | 막힌 순간 바로 보인다. |
| 대게 지사장 카드 내부 | `결재도장 광고` | 대게 1마리 이상 | 하위 생산자 자동화 체감과 직접 연결된다. |
| 전리품 조개 패널 | `조개 퀵배송 광고` | 캡슐 진행 중, 남은 시간 60초 초과 | 기존 5분 타이머와 궁합이 좋다. |
| 허세 진주 연구소 | `광고 연구 슬롯` | 연구 비용 부족 | 광고를 연구 대체재가 아니라 임시 땜빵으로 보이게 한다. |

## 4. 헤파이스토스 작업 큐

1. `config`에 4티어 생산자 정의 추가: 인턴/새우/대게/고래를 `costGrowth: 1.15`로 통일.
2. 경제 엔진에 `producesProducer` 타입 추가: 자원 생산과 생산자 생산을 분리.
3. 저장 데이터에 `producerProgress` 추가: 소수점 새우/대게 누적 보존.
4. 미션 추가: `대게 지사장 1마리`, `새우 자동 생산 5마리`, `고래 대주주 첫 영입`.
5. 광고 버프 상태 추가: `activeBuffs`, `cooldowns`, `lastVerifiedRewardId`.
6. Capacitor 패키징 브랜치 생성: 웹 빌드와 네이티브 싱크 루프만 먼저 안정화.
7. AdMob은 테스트 광고 ID로만 연결: SSV 원장 없이 실광고 붙이지 말 것.
8. 백엔드 원장 붙인 뒤 실광고 전환: 서명 검증, 멱등성, 쿨다운 로그가 먼저다.

## 5. 출시 전 숫자 QA 기준

| 검증 항목 | 합격 기준 |
|---|---|
| 초반 60초 | 인턴 최소 3개 이상 구매 가능 |
| 3분 | 새우 2~4마리 도달 가능, 미션 보상 최소 1회 |
| 5분 | 대게 가격이 보이지만 즉시 연속 구매는 불가 |
| 광고 1회 후 | 2분 안에 다음 의미 있는 구매 1회 발생 |
| 광고 미시청 | 90~150초 기다리면 같은 목표 도달 가능 |
| 대게 1마리 | 새우 자동 생산 진행률이 UI에서 감지 가능 |
| 오프라인 4시간 | 고래/대게 체인이 과도하게 폭발하지 않음 |

최종 방향: 이 게임은 착한 힐링 방치형이 아니다. `사장 말미잘을 때렸더니 조직도가 증식하고, 5분 뒤 광고 국물이 회사를 살리는 이상한 스타트업 시뮬레이터`가 되어야 한다.
